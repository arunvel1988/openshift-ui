OpenShift Storage

In OpenShift, storage is essential for running applications that need to preserve data beyond the lifecycle of a container. Kubernetes and OpenShift provide a range of storage solutions for different use cases.

Key Storage Types in OpenShift:

  • Ephemeral Storage: Temporary storage tied to the pod's lifecycle; data is lost when the pod is deleted.
  • Persistent Volumes (PVs): Storage resources in the cluster that can be dynamically or statically provisioned.
  • Persistent Volume Claims (PVCs): Requests for storage by users; PVCs bind to PVs.
  • Storage Classes: Define different types of storage with parameters like performance or replication.
  • Dynamic Provisioning: Automatically creates PVs when PVCs are requested.

Storage in OpenShift can come from cloud providers (AWS EBS, Azure Disk, GCP Persistent Disk) or on-premises solutions like NFS, Ceph, or GlusterFS.
