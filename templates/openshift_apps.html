OpenShift offers a robust, Kubernetes-based platform for running containerized applications. Deployments in OpenShift are more than just "running containers" ‚Äî they bring in features like build automation, scaling, routing, and integrated security.

Key Concepts

  • Projects/Namespaces: Logical isolation for applications and resources.
  • BuildConfig: Defines how source code is built into an image.
  • DeploymentConfig / Deployment: Describes how pods are created and updated.
  • Services: Stable network endpoints for pods.
  • Routes: External access to services via HTTP/HTTPS.

Deployment Methods

  1. From Source Code: Use oc new-app with a Git repo to automatically build and deploy.
  2. From Container Images: Deploy prebuilt images directly from Docker Hub or an internal registry.
  3. Using Templates: Reuse pre-defined resource configurations.
  4. Helm Charts: Package and deploy complex apps with multiple components.

Example CLI Deployment

# Login to the cluster
oc login --token=YOUR_TOKEN --server=https://api.cluster.example.com:6443

# Create a new project
oc new-project myapp

# Deploy from source
oc new-app https://github.com/sclorg/nodejs-ex.git

# Expose the app to the internet
oc expose svc/nodejs-ex

# View the route
oc get routes

Best Practices

  • Keep configuration and secrets in ConfigMaps and Secrets.
  • Use resource limits to avoid noisy-neighbor issues.
  • Set readiness and liveness probes for health checks.
  • Automate builds and deployments with pipelines (Tekton/Jenkins).
